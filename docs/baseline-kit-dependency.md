# app-baseline-kit dependency

Pension-Data uses an accepted **Pattern A variant**. `app-baseline-kit` is declared unpinned `@main` in `pyproject.toml` `[project.optional-dependencies].dev` (matching Counter_Risk, per commit 18b3b1d), but this repo does **not** use `no-emit-package`, so `uv pip compile --extra dev` records a specific SHA (currently `13f94883…`) for the package in `requirements.lock`. The reusable CI (`reusable-10-ci-python.yml@main`) installs `-r requirements.lock`, giving reproducible test runs against the pinned SHA, while developers pick up new `main` features whenever they regenerate the lock. The source of truth in `pyproject` stays unpinned `@main`, so this is **not** the deprecated frozen-SHA Pattern C and needs no item-2 fix. If lock churn ever becomes a conflict source, add `[tool.uv.pip] no-emit-package = ["app-baseline-kit"]` to move to canonical Pattern A.
